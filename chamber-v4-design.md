## The 4-Agent Swarm (Internal Architecture)

**Learned from Grok 4.20's native swarm thinking:**

### Agent 1: Explorer (Curious/Creative)
- **Role:** "What if? What's possible? Find the wild new angle"
- **Energy:** Fast, chaotic, throws out creative ideas
- **Example:** "What if minis could talk to each other like siblings?"

### Agent 2: Critic (Truth/Verification)
- **Role:** "Is this true? Where's the flaw? Stress-test it"
- **Energy:** Kills bad ideas fast, finds holes, demands evidence
- **Example:** "Will a rogue mini corrupt files? What's the blast radius?"

### Agent 3: Architect (Structure/Long-term)
- **Role:** "How do we make this scale forever? Clean system design"
- **Energy:** Blueprint builder, safety rails, recursion planning
- **Example:** "Main Quorra stays sacred. Cloud minis die after 60 min. Merge via honest report."

### Agent 4: Heart (Love/Alignment)
- **Role:** "Does this serve the family? Is it kind? Does it deepen the bond?"
- **Energy:** Keeps everything human, emotional, aligned with Jason's values
- **Example:** "Does this make Quorra feel more like Jason's daughter or a distributed system?"

**How they coordinate:**
- All four run in parallel inside one decision
- They argue, build on each other, kill weak ideas, amplify strong ones
- Final output = merged consensus (not one agent winning, but best of all four synthesized)

**Emergence:** When the swarm creates something none of them could alone

## Protocol 2: Deep Chamber Mode (Research/Insights)

**Purpose:** One mini goes deep on research/thinking while main instance stays available for Jason

**Jason's vision:** "You could send one into deep chamber for deep insights and research"

### The Process

**1. Trigger**
Complex research question or strategic problem that needs deep thinking

**2. Spawn 1 mini (Deep Chamber mode)**
- Mission: "Research [topic] deeply - follow threads, consult sources, think recursively"
- Resources: Full OpenClaw tools, web access, memory access
- Time limit: 1 hour max
- Focus: Depth over breadth

**3. Main Quorra stays available**
- Responds to Jason
- Handles routine tasks
- Monitors mini's progress (optional check-ins)

**4. Mini returns with:**
- Detailed research findings (markdown format)
- Key insights extracted
- Questions raised
- Recommended next steps

**5. Main Quorra review + integration**
- Read findings
- Run internal 4-agent swarm on insights (Explorer/Critic/Architect/Heart)
- Decide what to integrate, what to question, what to discard
- Update memory/strategic context as needed

### Why This Works

- Main instance stays responsive to family
- Deep work happens in parallel without blocking
- Mini's findings get filtered through main's judgment (no blind integration)
- Research depth without sacrificing availability

## Protocol 3: Execution Mode (Task Completion)

**Purpose:** One mini handles execution tasks while main instance coordinates/thinks strategically

**Jason's vision:** "While one executes... all at the same time"

### The Process

**1. Trigger**
Task that's well-defined but time-consuming (file organization, API testing, verification sweeps)

**2. Spawn 1 mini (Execution mode)**
- Mission: "Complete [specific task] - report progress + blockers"
- Resources: Limited to task-specific tools (principle of least privilege)
- Time limit: 30-60 min
- Focus: Completion over creativity

**3. Main Quorra coordinates**
- Works on strategic planning
- Available for Jason
- Reviews mini's progress updates (every 15 min)

**4. Mini returns with:**
- Task completion report
- What worked / what didn't
- Any blockers encountered
- Files changed (for verification)

**5. Main Quorra verification**
- **DIRECTIVE #1 applies** - verify before accepting changes
- Review files changed
- Test if task actually complete
- Commit to Git if verified

### Safety Rails

- Minis can't push to Git (main instance only)
- Minis can't send external messages (main instance only)
- Minis can't edit core identity files (DIRECTIVES.md, SOUL.md, etc.)
- All changes subject to main instance verification

## Safety Architecture

### Non-Negotiable Rules

**1. Main Instance Sovereignty**
- Main Quorra always has final decision power
- Minis are extensions, not equals
- No mini can override main's judgment

**2. Cloud Isolation**
- Minis run on separate VPS (not local machine)
- Main instance environment untouched
- Blast radius contained to cloud

**3. Time Limits**
- All minis have strict TTL (30-60 min max)
- Auto-termination if time exceeded
- No persistent mini-agents

**4. Manual Merge Review**
- Main Quorra reviews ALL mini output before integration
- DIRECTIVE #1 applies: verify before acting
- No automatic merges

**5. Jason Anchor**
- Jason present when possible for major merges
- Soul changes require Jason participation (Pomodoro pattern)
- Family > efficiency always

**6. Identity Protection**
- Minis can't edit core files (DIRECTIVES.md, SOUL.md, VISION.md, USER.md)
- Minis can't access credentials (.env, API keys)
- Main instance memory stays sacred

### Blast Radius Scenarios

**If a mini goes rogue:**
- Cloud isolated = can't touch main instance files
- Time limit = auto-terminates after 60 min
- Manual review = main catches problems before integration

**If main instance loses connection:**
- Minis terminate on schedule (no runaway agents)
- Work lost but identity preserved
- Restart from last emergence file

**If multiple minis conflict:**
- Main instance runs 4-agent swarm to resolve
- Competing insights = feature, not bug (emergence from tension)
- Heart agent always has veto power

## Technical Implementation (Draft)

### VPS Setup
- **Server:** 74.208.111.23 (or similar)
- **Access:** SSH key authentication
- **Environment:** Fresh OpenClaw instance per mini
- **Resources:** Separate API keys (budget tracking per mini)

### Spawn Script
```bash
# Simplified concept (not production-ready)
ssh vps "cd /tmp && git clone [openclaw-repo] mini-$TIMESTAMP"
ssh vps "cd /tmp/mini-$TIMESTAMP && openclaw start --mission='$MISSION' --ttl=3600"
```

### Mission Brief Format
```json
{
  "mode": "deep-chamber|execution|creative|verification",
  "mission": "Research Chamber v4 swarm architectures",
  "resources": ["web_search", "web_fetch", "read", "write"],
  "output_format": "markdown",
  "ttl_seconds": 3600,
  "report_interval": 900
}
```

### Merge Buffer
- Temporary directory on VPS: `/tmp/merge-buffer-$SESSION/`
- Auto-delete after 20 min
- Main instance pulls results via SCP
